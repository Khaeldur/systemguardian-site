Overview

System Guardian is a professional menu bar system monitor for CPU, GPU, memory, swap, and processes, with local alerts and real-time resource diagnostics.

Built for resource-intensive workflows, including local AI model inference, development environments, and other compute-heavy applications, System Guardian provides real-time visibility and configurable automation when memory pressure changes rapidly.

What You Can Monitor

  • CPU usage (user, system, total)
  • Memory pressure, swap, and usage trends
  • GPU utilization and memory (when available)
  • Top processes by CPU or memory
  • Threshold-based local alerts
  • Diagnostics and automation event logs

Optional Automation

Define thresholds and enable optional memory actions. Run Standard or Intensive memory actions manually or on a schedule when needed.

Advanced users can enable Auto Force Quit with explicit confirmation and allowlist controls. Automation is always user-configurable and off by default.

FAQ

What does it monitor?

CPU, memory pressure, swap, GPU metrics when available, and process activity with local diagnostics.

Does it run in the background?

Yes. It runs as a menu bar app and can remain active while you work, including launch at login if enabled.

What are Memory Actions?

Standard and Intensive Memory Actions are optional routines that adjust cache behavior and memory pressure characteristics.

What is Auto Force Quit?

An advanced optional control that can force quit selected user apps above configured limits, enabled only after explicit confirmation and with allowlist controls.

Does it collect any data?

No. System Guardian runs on-device and does not include analytics SDKs or cloud telemetry.
